# 番茄专注计时器 UI 设计规范（V1）

## 1. 目标与范围
本规范用于约束首个版本（V1）UI实现，目标是与当前静态 Demo 保持一致。  
适用页面：`dashboard`、`settings`、`data`。

基准文件：
- `D:\test2\index.html`
- `D:\test2\styles.css`

## 2. 设计原则
1. 暖色系、低饱和、轻拟物层次，不使用高对比纯黑主题。
2. 卡片化信息组织，模块边界清晰，圆角统一。
3. 信息优先级通过字号、字重、颜色深浅区分，不依赖复杂装饰。
4. 首版只保留 demo 中已出现的视觉元素，不新增风格化组件。

## 3. 设计 Tokens

### 3.1 配色方案（Color Tokens）
| Token | 值 | 用途 |
|---|---|---|
| `--bg` | `#d9d8d5` | 页面背景基色 |
| `--panel` | `#efeeec` | 主容器底色 |
| `--card` | `#f6f5f3` | 卡片底色 |
| `--line` | `#e5e2de` | 边框/分割线 |
| `--text` | `#4f3a1f` | 主文字色 |
| `--muted` | `#8a775d` | 次级文字色 |
| `--accent` | `#d4912a` | 强调色（状态点） |
| `--accent-soft` | `#f3dfbf` | 强调色外光晕 |

固定辅助色（保持与 demo 一致）：
- 主按钮底色：`#b97a22`
- 主按钮文字：`#fffdf7`
- 柱状图渐变：`#b5782a -> #d5a45e`
- 环形图剩余段：`#dbd7d0`

### 3.2 圆角与阴影
| Token | 值 | 用途 |
|---|---|---|
| `--radius-xl` | `26px` | 外层 Panel 圆角 |
| `--radius-lg` | `20px` | 常规卡片圆角 |
| `--radius-md` | `14px` | 中小组件圆角 |
| `--shadow` | `0 20px 50px rgba(28,24,18,0.08)` | 主容器阴影 |

## 4. 字体规范

### 4.1 字体家族
- 中文/英文主字体：`Outfit`, `"PingFang SC"`, `"Noto Sans SC"`, `sans-serif`
- 数字与计时字体：`"IBM Plex Mono"`, `ui-monospace`, `Consolas`, `monospace`

### 4.2 字重规范
- Regular：`400`
- Medium：`500`
- Semibold：`600`
- Bold：`700`（仅大标题可用）

## 5. 字体层级（Typography Scale）
| 层级 | 样式 | 说明 |
|---|---|---|
| Hero 标题 | `clamp(34px, 4.1vw, 54px) / 1 / 600` | Dashboard 主标题 |
| 页面标题 | `24px / 1.2 / 600` | 设置页、数据页标题 |
| 卡片标题 | `20px / 1.2 / 500~600` | 今日专注/本周专注等 |
| 正文大号 | `18px / 1.4 / 400` | Hero 副文案 |
| 正文标准 | `16px / 1.4 / 600` | 关键数据值 |
| 按钮文字 | `14px / 1 / 600` | 所有按钮 |
| 标签文字 | `12px / 1.2 / 500~600` | chip/mode/周卡标注 |
| 辅助提示 | `13px / 1.4 / 400~500` | hint/muted |
| 计时数字 | `clamp(48px, 7vw, 80px) / 1 / 600`（Mono） | 主倒计时 |
| 比例数字 | `28px / 1 / 600`（Mono） | 环形图中心百分比 |

## 6. 布局与栅格

### 6.1 容器
- 页面主容器 `.shell`：`max width 1200px`，左右留白 `24px`（桌面），`12px`（移动端）
- `.shell` 内边距：`22px`（桌面），`14px`（移动端）

### 6.2 Dashboard 布局
- 主栅格：`2fr 1fr`，间距 `14px`
- 小屏（<=980px）：切换为单列

### 6.3 页面切换
- `.page` 默认隐藏，`.page.active` 显示
- 顶部 `dashboard/settings/data` 按钮只负责页面显隐切换

## 7. 卡片样式规范

### 7.1 通用卡片（`.card`）
- 背景：`var(--card)`
- 边框：`1px solid var(--line)`
- 圆角：`20px`
- 内边距：`16px`

### 7.2 Hero 卡片
- 最小高度：`420px`（小屏 `360px`）
- 计时信息容器 `.hero-timer`：圆角 `18px`，边框 `1px`，背景 `#f8f7f5`

### 7.3 数据卡片
- `today/timeslot/weekly` 最小高度：`203px`
- 周格子卡片：圆角 `10px`，内边距 `8px`

## 8. 组件样式规范

### 8.1 顶部导航 Pill（`.pill`）
- 圆角：`999px`
- 内边距：`8px 14px`
- 默认：浅底+次级字色
- 激活态：文字切换为主色，边框加深（`#d8d5d0`）

### 8.2 Chip（`.chip`）
- 圆角：`999px`
- 字号：`12px`，字重 `600`
- 背景：`#f1efec`
- 边框：`1px solid var(--line)`

### 8.3 按钮（`.btn`）
- 高度由内边距控制：`10px 14px`
- 圆角：`12px`
- 边框：`1px solid #d9d5ce`
- 默认按钮：浅底、主色字
- 主按钮（`.btn-dark`）：暖黄棕底 `#b97a22`，浅色字 `#fffdf7`
- 全宽按钮（`.full`）：`width: 100%`

### 8.4 输入框占位样式（`.fake-input`）
- 用于静态展示：圆角 `11px`，高度 `42px`，浅底+细边框

### 8.5 开关（`.toggle`）
- 尺寸：`56x32`
- 未开启：底色 `#ddd9d3`
- 开启：底色 `#b97a22`
- 圆点：`24x24` 白色，轻阴影

### 8.6 状态提示（`.status`）
- 容器：浅底边框卡片
- 状态点：`10x10`，`var(--accent)`，外扩光晕 `0 0 0 8px var(--accent-soft)`

### 8.7 图表（静态）
- 环形图：`128x128`，`conic-gradient(var(--text) 0 78%, #dbd7d0 78% 100%)`
- 柱状图：高度 `110px`，柱圆角 `8px 8px 3px 3px`，暖棕渐变

## 9. 间距规范（Spacing）
- 主模块间距：`14px`
- 小组件间距：`8px`
- 文本组常用底部间距：`12px`
- 表单行间距：`11px`

## 10. 响应式规范

断点：
- `<=980px`：Dashboard 由双列变单列；hero-timer 双列变单列
- `<=640px`：顶部栏改为纵向，导航按钮等宽分布，标题字号下降

移动端保持原则：
1. 不改变组件视觉语言（颜色、圆角、边框保持一致）
2. 只调整布局结构与字号，不重做组件

## 11. 交互状态规范（V1 静态）
1. 页面级：`dashboard/settings/data` 可点击切换可见页面
2. 按钮级：视觉区分默认/激活，不定义复杂 hover 动效
3. 表单级：仅做静态展示，不实现输入、校验、提交逻辑

## 12. 开发落地约束（保证与 Demo 一致）
1. 首版不得替换字体、主色、圆角体系。
2. 首版不得新增玻璃拟态、强渐变背景、深色主题分支。
3. 首版新增组件必须复用现有 token（颜色/圆角/边框/字号）。
4. 视觉回归以 `index.html + styles.css` 为唯一验收基线。

## 13. 验收清单（UI）
1. 三个页面点击切换正常，激活按钮状态正确。
2. 主页面四卡片结构完整且层级一致。
3. 所有按钮样式统一，主按钮为暖黄棕。
4. 文本层级符合字体规范，计时数字使用等宽字体。
5. 移动端断点下无布局重叠、无文字截断。
