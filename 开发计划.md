# 番茄专注计时器 V1 开发计划（基于 PRD + 静态 Demo）

## 0. 项目目标（统一上下文）
- 项目目标：把当前静态 Demo 演进为可用的 Web 产品 V1，满足 PRD 的 P0 与核心 P1（数据导入导出）。
- 必须保留：暖色系视觉、页面结构、组件样式与 `D:\test2\index.html` + `D:\test2\styles.css` 一致。
- 关键功能：25/5 番茄钟、自定义时长、提示音、今日/本周/时段统计、本地存储、数据导入导出。

## 2. 里程碑与步骤（按依赖顺序）

### 步骤 1：工程初始化与模块化入口
- 本步目标：把当前单页静态文件升级为“可维护的模块化前端工程”。
- 前置依赖：无。
- 涉及文件：
  - 新增：`package.json`、`vite.config.js`（可选）、`src/main.js`
  - 修改：`index.html`（脚本入口改为 `src/main.js`）
- 具体实现：
  - 使用 Vite 初始化 Vanilla JS 项目。
  - 保留现有 DOM 结构和样式文件，不改视觉。
  - 页面可正常启动开发服务并加载。
- 完成标准（DoD）：
  - `npm run dev` 可启动。
  - 页面与当前 Demo 视觉一致。
  - 顶部导航切页功能仍可用。

### 步骤 2：路由切页逻辑独立（Dashboard/Settings/Data）
- 本步目标：把页面切换从内联脚本迁移到可复用模块。
- 前置依赖：步骤 1。
- 涉及文件：
  - 新增：`src/app/router.js`
  - 修改：`src/main.js`、`index.html`（移除内联脚本）
- 具体实现：
  - 抽象 `activatePage(pageName)` 与按钮绑定逻辑。
  - 统一管理 `data-page` / `data-page-btn` 的激活状态。
- 完成标准（DoD）：
  - 所有导航按钮与页面内跳转按钮可点击并正确切页。
  - `index.html` 不再包含页面切换内联 JS。

### 步骤 3：本地数据模型与存储层（P0 基础）
- 本步目标：定义唯一数据结构，打通 localStorage 读写。
- 前置依赖：步骤 2。
- 涉及文件：
  - 新增：`src/data/schema.js`、`src/data/storage.js`
  - 新增：`src/timer/timerState.js`
  - 修改：`src/main.js`
- 具体实现：
  - 定义 `AppState`：计时配置、当前计时状态、历史会话、统计缓存。
  - 实现 `loadState()` / `saveState()` / `resetState()`。
  - 首次访问自动写入默认配置（25/5、提示音开启）。
- 完成标准（DoD）：
  - 刷新页面后设置与状态可恢复。
  - localStorage 中存在结构化对象（含版本号 `schemaVersion`）。

### 步骤 4：番茄钟核心引擎（不接 UI）
- 本步目标：完成计时状态机，独立可测试。
- 前置依赖：步骤 3。
- 涉及文件：
  - 新增：`src/timer/timerEngine.js`
  - 修改：`src/timer/timerState.js`
- 具体实现：
  - 支持：`start`、`pause`、`resume`、`reset`、`tick`。
  - 模式切换：`focus -> break -> focus`。
  - 结束时触发 `onSessionComplete` 事件。
- 完成标准（DoD）：
  - 25 分钟结束后自动切换休息时段。
  - 暂停后时间不走；恢复后继续。
  - 重置回当前模式初始时长。

### 步骤 5：主页面计时交互接入（P0）
- 本步目标：把计时引擎接到 Dashboard 的按钮和展示。
- 前置依赖：步骤 4。
- 涉及文件：
  - 新增：`src/ui/dashboardView.js`
  - 修改：`index.html`（补充稳定选择器如 `id` 或 `data-role`）
  - 修改：`src/main.js`
- 具体实现：
  - 将倒计时文本、模式标签、轮次显示绑定真实状态。
  - 按钮行为：开始/暂停/重置。
  - 页面激活时也能保持最新显示。
- 完成标准（DoD）：
  - 点击开始后秒级倒计时。
  - 点击暂停后停住，再点继续恢复。
  - 重置后恢复当前模式完整时长。

### 步骤 6：设置页面功能（专注/休息时长、提示音开关）（P0）
- 本步目标：实现设置可编辑、可保存、即时生效。
- 前置依赖：步骤 3、步骤 5。
- 涉及文件：
  - 新增：`src/settings/settingsService.js`、`src/ui/settingsView.js`
  - 修改：`index.html`（把静态输入改为可输入控件）
  - 修改：`src/main.js`
- 具体实现：
  - 支持修改专注时长与休息时长（分钟）。
  - 支持提示音开关。
  - 保存后写入 storage，并影响后续计时。
- 完成标准（DoD）：
  - 新时长保存后刷新不丢失。
  - 新建下一轮番茄时按新配置执行。
  - 提示音开关状态可持久化。

### 步骤 7：计时结束提示音（P0）
- 本步目标：计时完成时播放提示音，受设置项控制。
- 前置依赖：步骤 6。
- 涉及文件：
  - 新增：`assets/sounds/timer-done.mp3`
  - 新增：`src/app/events.js`（或 `src/app/audio.js`）
  - 修改：`src/timer/timerEngine.js`、`src/main.js`
- 具体实现：
  - 监听 session 完成事件。
  - 若 `soundEnabled=true` 则播放音频。
  - 播放失败不阻塞主流程。
- 完成标准（DoD）：
  - 专注或休息结束均可触发提示音。
  - 关闭提示音后不播放。

### 步骤 8：专注记录落库（会话日志）（P0）
- 本步目标：每次有效专注完成后形成可统计数据。
- 前置依赖：步骤 5、步骤 6。
- 涉及文件：
  - 修改：`src/timer/timerEngine.js`、`src/data/storage.js`
  - 新增：`src/stats/statsService.js`
- 具体实现：
  - 完成一轮专注后写入 `sessions[]`（开始时间、结束时间、时长、模式）。
  - 定义“有效专注”的记录条件（建议：专注时段完整完成）。
- 完成标准（DoD）：
  - 完成番茄后 localStorage 中 `sessions` 数组新增一条记录。
  - 刷新页面后记录仍在。

### 步骤 9：今日专注/本周专注/时段统计渲染（P0 + 核心展示）
- 本步目标：Dashboard 三块统计改为真实数据。
- 前置依赖：步骤 8。
- 涉及文件：
  - 修改：`src/stats/statsService.js`
  - 修改：`src/ui/dashboardView.js`
- 具体实现：
  - 今日统计：总时长、完成番茄数、目标占比（可基于配置目标）。
  - 本周统计：按周一到周日聚合。
  - 时段统计：按小时聚合映射到柱高。
- 完成标准（DoD）：
  - 完成若干轮后，三块统计数值会变化且刷新后一致。
  - 今日/本周边界按本地时区计算正确。

### 步骤 10：数据导出功能（P1）
- 本步目标：支持将本地数据导出为文件备份。
- 前置依赖：步骤 3、步骤 8。
- 涉及文件：
  - 新增：`src/data/importExport.js`
  - 新增：`src/ui/dataView.js`
  - 修改：`index.html`、`src/main.js`
- 具体实现：
  - 点击“导出数据”下载 JSON 文件（含 schemaVersion 与导出时间）。
  - 数据来源为当前完整 `AppState`。
- 完成标准（DoD）：
  - 点击导出可下载 `.json` 文件。
  - 文件可读且包含关键字段（settings/sessions）。

### 步骤 11：数据导入功能（P1）
- 本步目标：支持从导出文件恢复数据并反馈结果。
- 前置依赖：步骤 10。
- 涉及文件：
  - 修改：`src/data/importExport.js`、`src/data/schema.js`
  - 修改：`src/ui/dataView.js`
- 具体实现：
  - 文件选择后校验 JSON 结构与版本。
  - 校验通过则覆盖写入 localStorage，并刷新页面状态。
  - 失败时展示明确错误文案（格式错误/版本不支持）。
- 完成标准（DoD）：
  - 导入合法文件后统计和设置同步恢复。
  - 导入非法文件会失败且不会污染现有数据。

### 步骤 12：UI 一致性收口与验收（首版发布前）
- 本步目标：确保功能可用且样式严格对齐 Demo 与 UI 规范。
- 前置依赖：步骤 1 至 步骤 11。
- 涉及文件：
  - 修改：`styles.css`（仅必要修正）
  - 对照：`D:\test2\UI设计规范.md`
  - 可新增：`README.md`（运行说明）
- 具体实现：
  - 对照 UI 规范逐项核对（颜色、字号、圆角、间距、按钮状态）。
  - 手工回归三页面关键路径：计时、设置、导入导出。
  - 增加最小发布说明。
- 完成标准（DoD）：
  - P0 全通过，P1（导入导出）通过。
  - 视觉与静态 Demo 一致，无明显偏差。
  - 新人按 README 可本地启动与验证。

## 4. 项目技术栈总览（V1）
- 前端框架：`Vanilla JavaScript`（不引入 React/Vue，降低复杂度，保证与 Demo 一致）
- 构建工具：`Vite`
- 样式方案：`原生 CSS` + 设计 Token（已在 `styles.css`）
- 数据存储：`localStorage`
- 数据交换：`JSON`（导入/导出）
- 音频能力：`HTMLAudioElement`
- 部署方式：
  - 首选：`Vercel`（零配置部署 Vite 静态站点）
  - 备选：`Netlify` 或 `GitHub Pages`

## 5. 建议实施顺序摘要
1. 先完成步骤 1-5（让计时主流程可跑）。
2. 再完成步骤 6-9（让设置与统计闭环）。
3. 最后完成步骤 10-12（导入导出与发布验收）。
